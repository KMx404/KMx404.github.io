Programming - How to program better

Programming
Published on 16 Sep 2020
Learn what's wrong with your code !

Note:

Some of the main Tips and notes are based on personal experience. I hope they may came in handy.




Sometimes while working on some project with a team or even in some company, and you would receive a code review saying that your code is "bad"? or maybe its not qualified to be considered "good". As a junior developer or any other beginner out there who just scrumbled into the world of programming. Here are some tips that may help you code better. And create some worthy scripts. ⚡


About good and bad code 💭

Good and Bad are subjective concepts that usually underlined an specific set of values that vary from project to project, even from team to team. When someone is telling you that your 'code is bad' it is not an universal truth, you should think on this way that what he/she wanted to tell was that your 'code is not what he/she would consider to be good code' 🙄

You should always need to ask why your code is not covering the expectations. You need to adapt your code to your team values, if those values are already defined if not you need to encourage the discussion to figure out what is the correct way to do things otherwise it will become a complete mess down the road. 🙃

What is wrong? 💢

You can't tell someone that their code is bad according to your own agenda but there are some universal characteristics of 'good' quality code. Normally you won't hear this in most on the online courses or university courses about CS.

Personally, when i started coding in clubs and online organization I met a lot of smart people who genuinely considered as "good" programemrs and they gave me some tips and advices. I dive down into DEV.TO and saw some good well-explained articles that may help you understand the technical issues.

Professional developers learn about these concepts by reading books 📚 about programming and learning from experience the concepts and methodologies used in different companies. It is people who are serious in the business of software who are interested in these concepts. The next list outline what, I believe, is a set of broadly accepted characteristics that moderate the quality of code:

  1. S.O.L.I.D principles
  2. Code Smells
  3. Refactoring
  4. Design patterns
  5. Separation of concerns
  6. Abstraction
  7. Propper abstraction

You need to understand the concepts listed above, if you want to become a real professional 'good' developer.

The are a lot of stuff that you will gain by experience. And practice for sure.

So CODE, CODE AND CODE!


The Aspects of good code 🔥

There are a lot of features that come along with code codes. I am gonna walk you through some of them, Feel free to follow !


  • Simple:

    The first sample that works is not the best version of your code. You always need to make your code simpler. Simpler in a way that anyone else with basic coding skills can get along. Simpler means more readable. 🔟

    A quote said by some guy 😅. ```A good code, is a simple code but you need to be a genius to understand the simplicity```

  • Readable:

    One of the most important quality of a code is how easily you can read it. A top notch code 🔥 is a one that makes it easy for other programmers to understand what you have written with a minimum amount of time and effort. A good naming strategy, smart use of comments and clear control flow are the vital ingredients that make your code easy to read and hence easier to maintain, augment or reuse.

  • Easy to maintin

    No matter how well you have written the code, there will be bugs 🐞, which you will have to fix. It is the easiness with which it can be fixed that will make it a well-maintained code. And, this all depends upon factors like readability, simplicity and modifiability. So, if you can maintain your code well, it will symbolize its high quality.

  • Efficient

    Write a code that runs fast and uses minimum system resources. This will account for the efficiency of your written code and will lead to faster execution times. 🔆




Those are some Notes, Tips for you to write better code. Next time you face your monitor do your best writing a top notch code and work with those advices. 😉
