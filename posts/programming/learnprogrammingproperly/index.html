Things to put in mind

There are few things you should put in mind before following any roadmap towards the goal of learning something.


There's no magical trick or some Specific path to follow in order to become a developer/programmer or to actually become good at coding! A lot of people walked on different roads and end up crossing the same goal. What you're about to read is method or kinda a roadmap that helped me learn programming. You're totally free to follow or skip!

A lot of people came to me and ask about some problem and if I didn't have ha solution the first thing I do either i google it or look on Stackoverflow, Github, W3school Even the good ones will need to google something at some point. And maybe the good ones are the ones who actually google more!

LET'S DIVE IN

Learn from the origins

Since i started learning how to code, a lot of people adviced me to follow the original documentations and tutorials since a lot of content creators / teachers out there miss some important stuff which will only hurt the student more!

Documentation is not only a way to track students’ learning and thinking but also a resource that should have an effect over future planning. When done correctly, documentation should take the viewer on a journey highlighting their development and the challenges they faced on the way. A documentation of their thinking process as well as their learning experiences.

« A well written documentation is better than any tutorial! » - said by @KMx404

You can easily find the original documentation on the original website of the language or tool you're aiming to learn!

Here's another website you can check! It give you light documentaions in 1page to a lot of programming langugaes. Not the original one but it simplify it for beginners Check Here!


Note: You can check few tutorials from now and then. but make sure not to fall in some tutorial purgatory!


Practice practice and practice...

You might be surprised of How obvious are the steps I am mentioning here but make sure that those advices/methods led a lot of people to become good programmers.


After learning the basics and having a basic idea in mind about the things you're learning. Its time to pratice. As a competitive programmer I should mention participating in some online contests like the ones on codeforces or HackerRank.

Diving in competitive programming will for sure help you practice Algorithms and Data structures but also getting used on the programming language you're learning or aiming to learn.

Check HackerRank it is an Amazing website made for you to practice on multiple programming languages and sets of tools to learn with easy - editorial problems.

There's also CodeForces One of the top websites for online and worldwide competitions

Work on some projects!

There is no better way to actually learn something and be good in it other than working on some projects.

If you're wondering which project to work on, you can easily ask someone who have enough knowledge to guide you!

Don't try some advanced projects at first. Make sure you start from the beginning and build things from scratch.

Another thing that might help is creating an agenda of some projects you need to work once you complete a specific skill. The to-do list needs to be in order from the simplest projects to the most advanced one.



The good programmer needs to be a learning machine in order to keep with this fast evolution. While learning new things try to come up with your own methods and techniques that can make the learning path more easier!
