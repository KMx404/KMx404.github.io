How can you choose your web stack?

Note:

✋What am about to show you is a few examples and tips that `may` help you choose your webstack. You're not obliged to follow any of my tips!




What's a software stack and web stack?

In computing, a software stack is a set of software subsystems or components needed to create a complete platform such that no additional software is needed to support applications. Applications are said to "run on" or "run on top of" the resulting platform.

A web stack, also called a web application stack, is a compilation of software set up especially for implementing websites and web applications. For example, to develop a web application the architect defines the stack as the target: operating system, web server, database, and programming language.


We're going to divide the web stacks into two parts the `Front-end stack` and the `Back-end stack`

We're going to start with the Front-end and in the next article, we'll talk about the back-end stack

Front-end Stack

Talking about web development stacks that may differ from a framework to the other, But there are some basic technologies that present the web as we know it:

  • HTML
  • CSS
  • JavaScript
  • SCSS
  • SASS

What i just mentioned are just the basics core of the front-end stack Any modern website should contain some CSS,SCSS and JavaScript. What varies are the used libraries and Frameworks. Which we're going to talk about Next.


How do I choose a front-end framework?

It is hard to find a suitable framework for your use. Specially when you have no idea How to pick?

What should you consider when you're trying to choose a front-end framework?:

  • The License:

    Before you start developing with a framework, see what kind of license the framework is distributed under. While most licenses are pretty liberal to work with and lets you create commercial applications, some of them are not so generous. The last thing you'd want is to create an entire application only to find out that the license doesn't allow you to distribute it commercially. Better do your research before than suffering after.

  • The Community:

    Inevitably, even with proper documentation, you are going to run into errors to rectify which, you are going to have to ask the community behind the framework for help. I've personally interacted with communities which vitriolically attacked programmers new to the framework and sneered at them while on the other extreme I've seen communities cheerfully welcomes newbies and teach them the tricks of the trade.

  • Its Documentations:

    A framework's documentation is often key to its success. Well explained, detailed documentation draws in the power users and evangelists who then bring in more people. With shoddy, confusing documentation people are going to just walk off confused and annoyed.

  • Software Pattern:

    You need to put your mind into the Sofware pattern. Almost every framework out there execlusively uses the MVC pattern.which stands for Model-View-Controller, helps you keep your data: the model, the logic: the controller and the user interface: the view, separate from each other. This in turn lets you write better, tighter code which ultimately results in better applications.

    Just because almost everyone uses MVC doesn't mean that is everything you need to know though. There are a couple of variants including MVP: Model-View-Presenter, MVA: Model-View-Adapter and AVC: Application-View-Controller.

  • Hosting Requirements:

    As web developers, we may be inclined to building applications on cutting edge platforms but often the need and budget of the client comes first. Often it may be out of the budget to get a dedicated host to place our applications on and we'll have to settle with shared hosting with normal modules and settings.

  • Test to know!

    You can't fully know that your decision is well-based unless you can really test the Framework even for a little while Before diving in..

    Taking a consideration all what I just said earlier and build your final mind conculded by the test you make.
